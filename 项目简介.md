# IndustryVisionKit

## 项目简介

IndustryVisionKit 是一个基于 C++ / Qt Widgets / CMake 的通用检测系统示例工程，包含登录注册、模型管理、图片/视频/摄像头三种输入模式、检测结果展示、结果导出和实时日志等功能。

项目按职责拆分为两个子项目：

- `IndustryVisionLib`：算法层与数据层，负责用户管理、检测数据结构、YOLO 统一接口与推理逻辑封装。
- `IndustryVisionGUI`：界面层，负责登录页、主检测页、界面切换、结果展示、日志与导出。

## 技术栈

- C++17
- Qt6 Widgets
- CMake 3.21+
- 纯代码界面构建，不使用 `.ui`

## 架构设计思路

### 1. 分层设计

- GUI 层只负责交互与展示，不直接耦合检测细节。
- Lib 层统一管理模型参数、检测结果与用户信息持久化。
- 通过 `YoloEngine` 统一封装 YOLOv5 / v7 / v8 / v9 / v11 / v26 的接入入口，便于后续替换为真实 ONNX Runtime、TensorRT 或 OpenCV DNN 后端。

### 2. 页面切换

- `ApplicationWindow` 使用 `QStackedWidget` 管理登录页与主检测页。
- 登录成功后切换到主检测界面，未登录时无法进入检测页。

### 3. 检测流程

- 模型路径 + YOLO 版本选择
- 置信度 / IOU 参数设置
- 选择图片 / 视频 / 摄像头输入源
- 执行检测并刷新原图、结果图、结果表格、汇总文本和日志
- 支持导出 `CSV/TXT`

## 当前实现说明

- 已完成完整 GUI、交互逻辑、用户本地注册登录、检测结果导出、日志输出与统一算法接口。
- 已预留并接入可选 `ONNX Runtime` 后端：当工程配置了 ONNX Runtime 后，会直接对 `.onnx` 模型执行真实推理。
- 未配置 ONNX Runtime 时，系统会自动退回模拟检测模式，便于继续联调界面与流程。

## 如何运行

### 1. 环境准备

- 安装 Qt 6（含 Widgets 模块）
- 安装 CMake 3.21+
- 配置支持 C++17 的编译器

### 2. 构建示例

默认构建：

```bash
cmake -S . -B build -DCMAKE_PREFIX_PATH=/你的Qt目录
cmake --build build --config Release
```

启用真实 ONNX 推理：

```bash
cmake -S . -B build \
  -DCMAKE_PREFIX_PATH=/你的Qt目录 \
  -DINDUSTRYVISION_ENABLE_ONNXRUNTIME=ON \
  -DINDUSTRYVISION_ONNXRUNTIME_ROOT=/你的onnxruntime目录

cmake --build build --config Release
```

构建产物会输出到项目根目录下的 `bin/` 文件夹。

### 3. 使用说明

- 首次启动先注册账号，用户信息会保存在本地 `AppDataLocation/users.json`
- 登录成功后进入主检测界面
- 默认会尝试指向 `resource/models/yolov8n.onnx` 和 `resource/images/bus.jpg`
- 选择模型文件并加载
- 设置阈值参数
- 切换输入源模式并选择文件或连接摄像头
- 执行检测、查看结果、导出数据

## 资源说明

- `resource/models/`：当前已内置多个 YOLO ONNX 模型样例
- `resource/images/`：当前已内置多张测试图片
- `resource/classes/`：类别名称文件，会根据模型名自动匹配加载
